# Security Policy

This document describes how to report security issues for **The CodeVerse Hub** website and related repositories under the `TheCodeVerseHub` GitHub organization.

For the more user-friendly version rendered on the site, see the Security Notice page:

- https://codeversehub.dev/pages/security-notice (or `/pages/security-notice`)

---

## Supported Projects

This policy applies to:

- This repository: `cvh_web` (CodeVerse Hub website)
- Official bots and tools under the `TheCodeVerseHub` organization
- Any other first‑party projects where this file or the Security Notice is referenced

---

## Reporting a Vulnerability

**Please do NOT open public GitHub issues or pull requests for security vulnerabilities.**

Instead, contact us privately:

- Email: **contact@thecodeversehub.tech**
- Or DM a server Admin / Owner in the CodeVerse Hub Discord

When reporting, please include (where possible):

- A clear description of the issue
- The affected project, page, or endpoint
- Steps to reproduce
- A minimal, non‑destructive proof of concept
- Any relevant logs, screenshots, or environment details

We aim to **acknowledge** valid reports within **48 hours** and will keep you informed of our progress.

---

## Disclosure Policy

- Please give us reasonable time to investigate and patch the issue before any public disclosure.
- Avoid testing in ways that could impact other users (e.g. data loss, service disruption).
- Once the issue is fixed, we may credit you (with your consent) in the project changelog or a dedicated acknowledgements section.

---

## Out of Scope

The following are generally **out of scope** for rewards or urgent handling:

- Denial-of-service or volumetric attacks (DDoS, brute forcing, etc.)
- Missing security headers without a working exploit
- Best-practice suggestions without a concrete vulnerability
- Social engineering or phishing against maintainers or community members

---

## Thank You

Responsible disclosure helps keep CodeVerse Hub safe for everyone.  
We appreciate the time and effort you invest in reporting security issues responsibly.
